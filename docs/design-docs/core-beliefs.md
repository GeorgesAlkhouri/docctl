# DDES-0001 Core Beliefs for Agent-First Delivery

## Intent
Define repository-level principles that maximize reliable autonomous implementation.

## Beliefs
1. Humans set intent; agents execute implementation loops.
2. Architecture constraints are leverage multipliers, not bureaucracy.
3. Repository-legible context is mandatory for consistent agent behavior.
4. Guardrails should be mechanical where possible.
5. Fast iteration plus bounded validation outperforms manual perfectionism.

## Operational Consequences
- Keep docs discoverable via indexes.
- Encode recurring feedback as rules, scripts, or tests.
- Prefer deterministic contracts and reproducible checks.
